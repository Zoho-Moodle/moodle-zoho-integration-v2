Lesson 06

Data Synchronisation Services

خدمات مزامنة البيانات

This lesson explains the concept of synchronisation, the two data directions in this project, how each sync service works step by step, and how scheduled (automatic) sync is managed using APScheduler.

يشرح هذا الدرس مفهوم المزامنة، اتجاهَي تدفق البيانات في المشروع، كيف تعمل كل خدمة مزامنة خطوةً بخطوة، وكيف يُدار الجدول الزمني التلقائي للمزامنة باستخدام APScheduler.

1. What Does "Sync" Mean?

١. ماذا يعني "المزامنة"؟

Synchronisation (sync) means making two separate systems have the same data. Imagine you have a student record in Zoho CRM and a user account in Moodle — they are in two completely separate databases on different servers. Sync bridges them: when data changes in one system, the sync service detects the change and updates the other system to match.

المزامنة تعني جعل نظامين منفصلين يحتويان نفس البيانات. تخيّل: سجل طالب في Zoho CRM وحساب مستخدم في Moodle — كلاهما في قواعد بيانات منفصلة على سيرفرات مختلفة. المزامنة تجسر بينهما: عند تغيير البيانات في نظام واحد، تكتشف خدمة المزامنة التغيير وتُحدّث النظام الآخر ليتطابق.

Direction 1: Zoho → Local DB → Moodle
  Zoho has new/updated student
    → Fetch from Zoho API
    → Upsert into local SQLite (students table)
    → Create/update user in Moodle via Moodle API
    → Enrol student in correct Moodle courses

Direction 2: Moodle → Local DB → Zoho
  Student gets a grade in Moodle
    → Moodle plugin fires webhook
    → Backend stores grade in grades table
    → Grade sync service pushes grade to Zoho Contact

2. The Mapper Pattern — Translating Between Systems

٢. نمط المحوّل — الترجمة بين النظامين

Each system uses different field names for the same concept. Zoho calls the first name "First_Name"; Moodle's API uses "firstname" (lowercase). The mapper pattern is a simple function that translates one format into the other, keeping the rest of the code clean.

كل نظام يستخدم أسماء حقول مختلفة لنفس المفهوم. Zoho يسمّي الاسم الأول "First_Name"؛ Moodle يستخدم "firstname" (بأحرف صغيرة). نمط المحوّل هو دالة بسيطة تُترجم صيغة إلى أخرى، مما يُبقي بقية الكود نظيفاً.

# backend/app/integrations/zoho/mapper.py

def zoho_contact_to_student(zoho_record: dict) -> dict:
    """
    Map a Zoho Contact record to our internal Student schema.
    Input (from Zoho): {"id":"xxx","First_Name":"Ahmad","Last_Name":"Ali",...}
    Output (for DB):   {"zoho_id":"xxx","first_name":"Ahmad","last_name":"Ali",...}
    """
    return {
        "zoho_id":    zoho_record.get("id"),
        "first_name": zoho_record.get("First_Name", ""),
        "last_name":  zoho_record.get("Last_Name", ""),
        "email":      zoho_record.get("Email", "").lower().strip(),
        "student_id": zoho_record.get("Student_ID__c"),
        "program":    zoho_record.get("Program__c"),
    }

def student_to_moodle_user(student) -> dict:
    """
    Map our internal Student (SQLAlchemy model) to Moodle's create_user format.
    """
    return {
        "firstname": student.first_name,
        "lastname":  student.last_name,
        "email":     student.email,
        "username":  student.email.split("@")[0].lower(),  # e.g. "ahmad.ali"
    }

The mapper is pure Python — no database calls, no API calls. Easy to test in isolation and easy to update when field names change.

المحوّل كود Python نظيف — بدون استدعاءات قاعدة بيانات أو API. سهل الاختبار بشكل منفصل، وسهل التحديث عند تغيير أسماء الحقول.

3. Full Student Sync Flow — Step by Step

٣. تدفق مزامنة الطلاب الكاملة — خطوة بخطوة

# backend/app/services/sync/student_sync.py (simplified)

from app.integrations.zoho.client import zoho_client
from app.integrations.zoho.mapper import zoho_contact_to_student
from app.integrations.moodle.client import moodle_client
from app.infra.db.models.student import Student
from app.services.event_log import log_event

def sync_all_students(db):
    """
    Pull ALL records from BTEC_Students module in Zoho, sync each one to local DB and Moodle.
    """
    log_event(db, "sync_students", "started")

    # STEP 1: Fetch all BTEC_Students records from Zoho (with pagination)
    zoho_students = zoho_client.fetch_all("BTEC_Students")
    print(f"Step 1: Got {len(zoho_students)} records from BTEC_Students")

    created_count = 0
    updated_count = 0
    error_count   = 0

    for contact in zoho_students:
        try:
            # STEP 2: Translate Zoho fields to our internal format
            student_data = zoho_contact_to_student(contact)

            # STEP 3: Upsert into local SQLite database
            student = upsert_student(db, student_data)

            if student.moodle_id is None:
                # STEP 4a: Student not yet in Moodle → create them
                result = moodle_client.create_user(
                    first_name = student.first_name,
                    last_name  = student.last_name,
                    email      = student.email,
                    username   = student.email.split("@")[0].lower(),
                )
                moodle_id = result[0]["id"]
                # Save the Moodle ID back to our DB so we know next time
                student.moodle_id = moodle_id
                db.commit()
                created_count += 1
                print(f"  Created Moodle user for {student.email}: ID={moodle_id}")
            else:
                # STEP 4b: Student already in Moodle → just update fields if needed
                updated_count += 1

        except Exception as e:
            error_count += 1
            log_event(db, "sync_students", "error",
                      entity_id=contact.get("id"), error=str(e))
            print(f"  Error processing {contact.get('Email','?')}: {e}")
            continue  # skip this student, continue with the rest

    summary = f"Sync complete: {created_count} created, {updated_count} updated, {error_count} errors"
    log_event(db, "sync_students", "success", details=summary)
    print(summary)
    return {"created": created_count, "updated": updated_count, "errors": error_count}

الخطوات بالترتيب: (1) جلب كل جهات الاتصال من Zoho مع التقسيم الصفحي. (2) ترجمة كل سجل. (3) حفظ/تحديث في قاعدة البيانات المحلية. (4) إنشاء مستخدم في Moodle أو تحديثه. كل خطأ يُسجَّل لكن لا يوقف المزامنة الكاملة.

4. Enrollment Sync — Linking Students to Courses

٤. مزامنة التسجيلات — ربط الطلاب بالكورسات

After students exist in both Moodle and the local DB, the enrollment sync reads registration records from Zoho and enrols each student in their correct Moodle course(s).

بعد وجود الطلاب في كلا النظامين، تقرأ مزامنة التسجيلات سجلات التسجيل من Zoho وتُسجِّل كل طالب في كورساته الصحيحة في Moodle.

def sync_enrollments(db):
    """
    For each registration in Zoho, ensure the student is enrolled
    in the corresponding Moodle course.
    """
    registrations = zoho_client.fetch_all_registrations()

    for reg in registrations:
        zoho_student_id = reg.get("Contact.id")
        moodle_course_id = get_moodle_course_id_for_program(reg.get("Program__c"))

        # Find the student in our local DB using their zoho_id
        student = db.query(Student).filter(
            Student.zoho_id == zoho_student_id
        ).first()

        if not student or not student.moodle_id or not moodle_course_id:
            continue  # skip if we cannot resolve IDs

        # Check if already enrolled to avoid duplicate enrollments
        existing = db.query(Enrollment).filter(
            Enrollment.student_id == student.id,
            Enrollment.moodle_course_id == moodle_course_id
        ).first()

        if not existing:
            # Not enrolled yet → enrol them in Moodle
            moodle_client.enrol_student(student.moodle_id, moodle_course_id)
            # Record the enrollment in our local DB
            db.add(Enrollment(
                student_id=student.id,
                moodle_course_id=moodle_course_id,
                status="active"
            ))
            db.commit()

5. Grade Sync — Pushing Grades from Moodle to Zoho

٥. مزامنة الدرجات — دفع الدرجات من Moodle إلى Zoho

The grade sync service reads grades from Moodle's API, stores them in the local grades table, and then pushes unsynced grades to the matching Zoho Contact field.

خدمة مزامنة الدرجات تقرأ الدرجات من Moodle API، تخزّنها في جدول grades المحلي، ثم تدفع الدرجات غير المُزامَنة إلى حقل جهة الاتصال المناسبة في Zoho.

def sync_grades(db):
    """Pull all grades from Moodle, push un-synced ones to Zoho."""
    students = db.query(Student).filter(Student.moodle_id.isnot(None)).all()

    for student in students:
        # Fetch course completion + grade from Moodle for this user
        moodle_grades = moodle_client.call(
            "core_grades_get_grades",
            {"userid": student.moodle_id}
        )
        for course_grade in moodle_grades.get("items", []):
            grade_value  = float(course_grade.get("graderaw") or 0)
            course_id    = course_grade["courseid"]
            passed       = grade_value >= 40.0  # configurable pass mark

            # Upsert grade record locally
            grade = upsert_grade(db, student.id, course_id, grade_value, passed)

            # Push to Zoho if not already done
            if not grade.synced_to_zoho and student.zoho_id:
                zoho_client.post(f"BTEC_Grades/{student.zoho_id}", {
                    "Grade__c":       "Pass" if passed else "Fail",
                    "Grade_Value__c": grade_value,
                })
                grade.synced_to_zoho = True
                db.commit()

6. Scheduled Sync with APScheduler

٦. المزامنة المجدوَلة باستخدام APScheduler

Running sync manually every time would be impractical. APScheduler is a Python library that runs functions automatically on a schedule (like a cron job). It is configured in backend/app/core/scheduler.py.

تشغيل المزامنة يدوياً في كل مرة غير عملي. APScheduler مكتبة Python تُشغّل الدوال تلقائياً وفق جدول زمني (مثل cron job). يُضبط في backend/app/core/scheduler.py.

from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger

scheduler = AsyncIOScheduler()

# Sync students every day at 02:00 AM
scheduler.add_job(
    run_student_sync,
    trigger=CronTrigger(hour=2, minute=0),
    id="student_sync",
    replace_existing=True,
)

# Sync grades every 6 hours
scheduler.add_job(
    run_grade_sync,
    trigger=CronTrigger(hour="*/6"),
    id="grade_sync",
    replace_existing=True,
)

# In main.py lifespan:
@asynccontextmanager
async def lifespan(app):
    Base.metadata.create_all(bind=engine)
    scheduler.start()     # <-- scheduler starts with the server
    yield
    scheduler.shutdown()  # <-- scheduler stops when server stops

You can also trigger sync manually from the Admin Panel at http://localhost:8001/admin/sync using the "Sync Now" buttons — useful for testing or urgent updates.

يمكنك أيضاً تشغيل المزامنة يدوياً من لوحة التحكم على http://localhost:8001/admin/sync عبر زر "Sync Now" — مناسب للاختبار أو التحديثات العاجلة.

APScheduler jobs run inside the same Python process as the FastAPI server. No separate process, no cron tab needed. If the server restarts, the scheduler restarts automatically with it because it is started in the lifespan context manager.

وظائف APScheduler تعمل داخل نفس عملية Python مع سيرفر FastAPI. لا عملية منفصلة، لا cron tab. إذا أعدت تشغيل السيرفر، يُعاد تشغيل الجدولة تلقائياً.
