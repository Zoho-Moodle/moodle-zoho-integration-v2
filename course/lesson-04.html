Lesson 04

Zoho CRM Integration & OAuth 2.0

التكامل مع Zoho CRM ونظام OAuth 2.0

This lesson explains what Zoho CRM is, how the project authenticates with it using OAuth 2.0, and how it reads and writes data to Zoho API calls.

يشرح هذا الدرس ما هو Zoho CRM، كيف يتم التحقق من الهوية باستخدام OAuth 2.0، وكيف يقرأ المشروع البيانات ويكتبها عبر Zoho API.

1. What is Zoho CRM?

١. ما هو Zoho CRM؟

Zoho CRM is cloud software used by the institute to manage student academic records. This project uses 8 custom BTEC modules — not the standard "Contacts" or "Deals". Student enrichment data lives in BTEC_Students, payment records in BTEC_Payments, class schedules in BTEC_Classes, and so on. Our backend reads from these modules and syncs into Moodle.

Zoho CRM برنامج سحابي يستخدمه المعهد لإدارة سجلات الطلاب الأكاديمية. يستخدم هذا المشروع 8 وحدات BTEC مخصصة — وليس "Contacts" أو "Deals" القياسية. بيانات الطالب التكميلية في BTEC_Students، المدفوعات في BTEC_Payments، الجداول في BTEC_Classes، وهكذا. يقرأ الباك-إند من هذه الوحدات ويزامن البيانات إلى Moodle.

2. What is OAuth 2.0? (For Complete Beginners)

٢. ما هو OAuth 2.0؟ (للمبتدئين تماماً)

OAuth 2.0 is a secure login system for apps. Instead of giving your username and password to our backend, Zoho gives a special short-lived access token. Our backend uses this token to prove it is allowed to read/write your Zoho data. When the token expires (usually every hour), the backend automatically gets a new one using a refresh token.

OAuth 2.0 نظام تسجيل دخول آمن للتطبيقات. بدلاً من إعطاء اسم المستخدم وكلمة المرور، يمنح Zoho رمزاً مؤقتاً خاصاً يسمى access token. يستخدمه الباك-إند لإثبات صلاحيته. عندما ينتهي صلاحية الرمز (عادةً كل ساعة)، يحصل الباك-إند تلقائياً على رمز جديد عبر refresh token.

OAUTH 2.0 TOKEN FLOW
====================
1. Developer visits api-console.zoho.com
2. Creates a "Self Client" — gets: CLIENT_ID + CLIENT_SECRET
3. Generates a one-time AUTHORIZATION_CODE (valid 5 mins)
4. Exchanges code → gets REFRESH_TOKEN (valid for months)
   Store REFRESH_TOKEN in .env file!
5. When backend starts:
   REFRESH_TOKEN + CLIENT_ID + CLIENT_SECRET
        ↓  (POST to accounts.zoho.eu/oauth/v2/token)
   ACCESS_TOKEN (valid 1 hour)
        ↓
   All API calls: Authorization: Zoho-oauthtoken {ACCESS_TOKEN}
6. When ACCESS_TOKEN expires → auto repeat step 5
Never share your CLIENT_SECRET or REFRESH_TOKEN. They give full access to your Zoho account. Keep them only in your .env file, which is in .gitignore and never committed to GitHub.
لا تشارك CLIENT_SECRET أو REFRESH_TOKEN أبداً. يمنحان وصولاً كاملاً لحساب Zoho. احتفظ بهما فقط في ملف .env، الذي يوجد في .gitignore ولا يُرفع إلى GitHub.

3. Step-by-Step: Getting OAuth Credentials from Zoho

٣. خطوة بخطوة: الحصول على بيانات OAuth من Zoho

  1. Go to https://api-console.zoho.eu (use .eu if your Zoho account is in Europe)
  2. Click Add Client → choose Self Client (for server-to-server scripts)
  3. Copy the Client ID → paste as ZOHO_CLIENT_ID in .env
  4. Copy the Client Secret → paste as ZOHO_CLIENT_SECRET in .env
  5. Click Generate Code tab. In the Scope field enter:
    ZohoCRM.modules.ALL,ZohoCRM.settings.ALL
  6. Set Time Duration to 10 minutes. Click Create.
  7. You get a Grant Token (the Authorization Code). Copy it — it expires in 5-10 minutes!
  8. Open a terminal and run this command (replace placeholders):
    curl -X POST https://accounts.zoho.eu/oauth/v2/token \
  -d grant_type=authorization_code \
  -d client_id=YOUR_CLIENT_ID \
  -d client_secret=YOUR_CLIENT_SECRET \
  -d redirect_uri=https://www.zoho.com \
  -d code=YOUR_GRANT_TOKEN
  9. The response JSON contains "refresh_token": "1000.xxxx...". Copy it → paste as ZOHO_REFRESH_TOKEN in .env

اتبع الخطوات أعلاه بالترتيب. الخطوة 7 مهمة: الـ Grant Token ينتهي خلال دقائق، نفّذ الأمر في التيرمينال فوراً. بعد الحصول على refresh_token أضفه في .env وأعد تشغيل السيرفر.

The ZOHO_API_DOMAIN in .env is https://www.zohoapis.eu for EU accounts, or https://www.zohoapis.com for US/global accounts. Check your Zoho account datacenter in Zoho Settings → Data Centers.

الـ ZOHO_API_DOMAIN في .env يكون https://www.zohoapis.eu للحسابات الأوروبية. تحقق من منطقتك في إعدادات Zoho → Data Centers.

4. The ZohoClient Class — Auto Token Refresh

٤. كلاس ZohoClient — التجديد التلقائي للرمز

The file backend/app/integrations/zoho/client.py contains a ZohoClient class that handles all communication with Zoho. Its most important job is automatically refreshing the access token behind the scenes.

ملف backend/app/integrations/zoho/client.py يحتوي على كلاس ZohoClient يتعامل مع كل الاتصالات مع Zoho. أهم وظيفته: تجديد الـ access token تلقائياً في الخلفية دون تدخل يدوي.

import requests
from datetime import datetime, timedelta
from app.core.config import settings

class ZohoClient:
    def __init__(self):
        self.access_token = None
        self.token_expires_at = datetime.utcnow()  # expired by default → forces refresh

    def _refresh_token(self):
        """Get a new access token using the refresh token."""
        resp = requests.post(
            f"{settings.ZOHO_ACCOUNTS_DOMAIN}/oauth/v2/token",
            params={
                "grant_type":    "refresh_token",
                "refresh_token": settings.ZOHO_REFRESH_TOKEN,
                "client_id":     settings.ZOHO_CLIENT_ID,
                "client_secret": settings.ZOHO_CLIENT_SECRET,
            }
        )
        data = resp.json()
        self.access_token = data["access_token"]
        # Zoho tokens last 3600 seconds (1 hour); refresh 5 min early to be safe
        self.token_expires_at = datetime.utcnow() + timedelta(seconds=3500)
        print("Zoho access token refreshed.")

    def _get_headers(self) -> dict:
        """Return Authorization headers, refreshing token if expired."""
        if datetime.utcnow() >= self.token_expires_at:
            self._refresh_token()  # auto-refresh!
        return {
            "Authorization": f"Zoho-oauthtoken {self.access_token}",
            "Content-Type": "application/json",
        }

    def get(self, endpoint: str, params: dict = None) -> dict:
        """Send a GET request to Zoho API."""
        url = f"{settings.ZOHO_API_DOMAIN}/crm/v6/{endpoint}"
        resp = requests.get(url, headers=self._get_headers(), params=params)
        resp.raise_for_status()   # throws exception if HTTP error (4xx, 5xx)
        return resp.json()

    def post(self, endpoint: str, data: dict) -> dict:
        """Send a POST request to Zoho API."""
        url = f"{settings.ZOHO_API_DOMAIN}/crm/v6/{endpoint}"
        resp = requests.post(url, headers=self._get_headers(), json={"data": [data]})
        resp.raise_for_status()
        return resp.json()

# Single shared instance used throughout the backend
zoho_client = ZohoClient()

The pattern zoho_client = ZohoClient() at module level creates ONE instance shared by all services. The first API call triggers a token refresh; subsequent calls reuse it until it expires.

إنشاء نسخة واحدة zoho_client = ZohoClient() على مستوى الموديول يعني أن جميع الخدمات تشترك في نفس الاتصال. أول استدعاء API يُجدّد الرمز؛ الاستدعاءات التالية تعيد استخدامه حتى انتهاء صلاحيته.

5. Fetching Records from Zoho — Paginated API Calls

٥. جلب السجلات من Zoho — استدعاءات API المُقسَّمة

Zoho API returns at most 200 records per page. If there are 800 students, you need 4 API calls (page 1, 2, 3, 4). The sync service handles this automatically:

Zoho API يعيد كحد أقصى 200 سجل في كل صفحة. إذا كان عندك 800 طالب تحتاج 4 استدعاءات (صفحة 1, 2, 3, 4). خدمة المزامنة تتعامل مع هذا تلقائياً:

def fetch_all_btec_students() -> list[dict]:
    """Fetch ALL records from BTEC_Students module, handling pagination."""
    all_students = []
    page = 1
    per_page = 200   # Zoho maximum (200 records per page)

    while True:
        response = zoho_client.get(
            "BTEC_Students",   # <-- the actual custom module API name
            params={
                "page": page,
                "per_page": per_page,
                "fields": "id,Student,Profile_Image,Department,Student_ID_Number,Branch_ID,Status"
                # Only fetch the fields we actually need — saves API quota!
            }
        )
        records = response.get("data", [])
        if not records:
            break   # no more pages — stop

        all_students.extend(records)

        # Check if Zoho says there are more pages
        if not response.get("info", {}).get("more_records", False):
            break

        page += 1

    print(f"Fetched {len(all_students)} records from BTEC_Students")
    return all_students
Always specify the fields parameter! Fetching all 50+ Zoho fields when you only need 6 wastes API quota, memory, and time. Zoho gives you a limited number of API calls per day (the daily limit depends on your plan).

دائماً حدّد الـ fields! جلب كل 50+ حقل من Zoho عندما تحتاج 6 فقط يُضيّع حصتك اليومية من API والذاكرة والوقت.

6. Zoho Field Names — API Name vs Display Name

٦. أسماء حقول Zoho — الاسم التقني مقابل اسم العرض

In the Zoho web interface the field might be labelled "Student ID". But in the API it is called Student_ID__c (the __c suffix means it is a custom field). You must use the API name, not the display name, in all code.

في واجهة Zoho الرسومية قد يظهر الحقل باسم "Student ID". لكن في الـ API اسمه Student_ID__c (اللاحقة __c تعني حقل مخصص). يجب استخدام اسم API وليس اسم العرض في كل الكود.

To find API names: In Zoho CRM, go to Setup → Developer Space → APIs → API Names and find the module fields list.

لإيجاد أسماء الـ API: في Zoho CRM اذهب إلى Setup → Developer Space → APIs → API Names وابحث في قائمة حقول الوحدة التي تريدها.

7. Writing Data Back to Zoho

٧. كتابة البيانات مرة أخرى إلى Zoho

When a student's grade is updated in Moodle, the grade sync service pushes it back to Zoho. This updates the student's Contact record in Zoho CRM so the admin can see the grade without logging into Moodle.

عند تحديث درجات الطالب في Moodle، تدفع خدمة مزامنة الدرجات البيانات مرة أخرى إلى Zoho. هذا يُحدّث سجل الطالب في Zoho CRM ليتمكن المسؤول من رؤية الدرجة دون الدخول إلى Moodle.

def push_grade_to_zoho(zoho_id: str, grade_data: dict):
    """
    Update a Contact record in Zoho with grade information.
    zoho_id: the Contact's ID in Zoho (stored in our students table)
    grade_data: dict with grade_value, grade_letter, passed fields
    """
    payload = {
        "Grade__c":  grade_data["grade_letter"],   # e.g. "Merit"
        "Passed__c": grade_data["passed"],           # True or False
        "Grade_Value__c": grade_data["grade_value"], # e.g. 72.5
        "Last_Grade_Sync__c": datetime.utcnow().isoformat()
    }
    try:
        result = zoho_client.post(f"BTEC_Grades/{zoho_id}", data=payload)
        print(f"Grade pushed to Zoho BTEC_Grades record {zoho_id}")
        return result
    except Exception as e:
        # Log error but do not crash — grades are pushed on best-effort basis
        log_event("push_grade_to_zoho", "error", entity_id=zoho_id, error=str(e))
        raise
