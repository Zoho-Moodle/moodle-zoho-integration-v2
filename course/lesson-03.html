Lesson 03

Database Design with SQLite & SQLAlchemy

تصميم قاعدة البيانات باستخدام SQLite و SQLAlchemy

This lesson explains how the project stores its data — what a database is, what SQLite is, what an ORM is, and a walkthrough of all 10 tables in the project.

هذا الدرس يشرح كيف يخزّن المشروع بياناته — ما هي قاعدة البيانات، ما هو SQLite، ما هو الـ ORM، واستعراض لكل الجداول العشرة في المشروع.

1. What is a Database?

١. ما هي قاعدة البيانات؟

A database is a structured place to store data. Think of it like a collection of Excel tables — each table has columns (fields) and rows (records). A database lets you store, find, update, and delete data efficiently.

قاعدة البيانات هي مكان منظَّم لتخزين البيانات. تخيّلها كمجموعة من جداول Excel — كل جدول له أعمدة (حقول) وصفوف (سجلات). تسمح لك بتخزين البيانات والبحث فيها وتحديثها وحذفها بكفاءة عالية. في هذا المشروع، قاعدة البيانات تخزّن الطلاب، التسجيلات، الدرجات، المدفوعات، وسجل كل عملية مزامنة.

Table: students
+----+------------+-----------+---------------------+------------+
| id | first_name | last_name | email               | program    |
+----+------------+-----------+---------------------+------------+
|  1 | Ahmad      | Ali       | ahmad@example.com   | BTEC L3    |
|  2 | Sara       | Hassan    | sara@example.com    | BTEC L2    |
|  3 | Khalid     | Omar      | khalid@example.com  | BTEC L3    |
+----+------------+-----------+---------------------+------------+
Each row = one student record     Each column = one piece of information

2. What is SQLite? Why No Database Server is Needed

٢. ما هو SQLite؟ لماذا لا نحتاج سيرفر قاعدة بيانات؟

SQLite is a database that lives in a single file. No server process to start. No installation. No port number. No username or password. Just a file: moodle_zoho_local.db in the backend/ folder.

SQLite هو قاعدة بيانات تعيش في ملف واحد. لا يوجد سيرفر منفصل تحتاج لتشغيله. لا تثبيت إضافي. لا رقم بورت. لا اسم مستخدم أو كلمة مرور. فقط ملف واحد: moodle_zoho_local.db داخل مجلد backend/.

Analogy: If MySQL is a filing cabinet with a dedicated librarian (a separate server process), then SQLite is a notebook in your bag — always accessible, portable, no overhead.
تشبيه: إذا كان MySQL خزانة ملفات مع موظف مخصص (سيرفر)، فإن SQLite مثل دفتر ملاحظات في حقيبتك — دائماً متاح، محمول، بدون أي تعقيد.

The file is automatically created when your server starts for the first time. You never run a "CREATE DATABASE" command.

الملف يُنشأ تلقائياً عند تشغيل السيرفر للمرة الأولى. لا تحتاج أن تكتب أي أمر "CREATE DATABASE".

# After running "python start_server.py", verify the file was created:
ls backend/moodle_zoho_local.db
# You will see the file — its size grows as data is synced
Important — Back it up! The database is just a file. Deleting or corrupting it loses ALL synced data. Copy it regularly to a backup location.
مهم — احتفظ بنسخة احتياطية! قاعدة البيانات مجرد ملف. حذفه أو تلفه يعني ضياع جميع البيانات. انسخه بانتظام إلى مكان آمن.

3. What is SQLAlchemy? The ORM Concept

٣. ما هو SQLAlchemy؟ مفهوم الـ ORM

An ORM (Object Relational Mapper) is a tool that lets you work with a database using Python classes instead of writing raw SQL. SQLAlchemy is Python's most popular ORM.

الـ ORM (Object Relational Mapper) أداة تتيح لك التعامل مع قاعدة البيانات باستخدام كلاسات Python بدلاً من كتابة SQL مباشرة. SQLAlchemy هو الـ ORM الأكثر شعبية في Python.

Without ORM — you write raw SQL:
  cursor.execute("INSERT INTO students (first_name, email) VALUES ('Ahmad', 'a@b.com')")

With SQLAlchemy ORM — you write Python:
  student = Student(first_name="Ahmad", email="a@b.com")
  db.add(student)
  db.commit()
  
SQLAlchemy writes the SQL for you behind the scenes!

Benefits: you get Python code completion, type safety, easier joins, and protection from SQL injection attacks.

فوائد الـ ORM: اكتمال تلقائي في المحرر، أمان من هجمات SQL injection، كود أنظف وأسهل قراءةً وصيانةً، ولا تضطر لتعلم SQL متقدم.

The SQLAlchemy engine is set up in backend/app/infra/db/base.py:

from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase

# "sqlite:///" means use SQLite file-based database
# "./moodle_zoho_local.db" is the file path, relative to where server runs (backend/)
engine = create_engine(
    "sqlite:///./moodle_zoho_local.db",
    connect_args={"check_same_thread": False}
    # check_same_thread=False is needed because FastAPI uses multiple threads
)

class Base(DeclarativeBase):
    pass
# All our model classes will inherit from Base

The database session (connection) is managed per-request in backend/app/infra/db/session.py:

from sqlalchemy.orm import sessionmaker

# SessionLocal creates new database sessions
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    """
    FastAPI dependency — provides a DB session for each request.
    The 'finally' block ensures the session is ALWAYS closed,
    even if an error occurs mid-request.
    """
    db = SessionLocal()
    try:
        yield db        # hand the session to the endpoint function
    finally:
        db.close()      # always close when done

الـ session هو "الاتصال" بقاعدة البيانات. في FastAPI، نُنشئ session جديد لكل طلب HTTP ونُغلقه بعد انتهائه. هذا يمنع تسرّب الاتصالات.

4. How a Model (Table) is Defined in Code

٤. كيف يُعرَّف النموذج (الجدول) في الكود

Each database table is defined as a Python class that inherits from Base. Here is what the Student model looks like (simplified):

كل جدول في قاعدة البيانات يُعرَّف ككلاس Python يرث من Base. إليك شكل نموذج Student المبسط مع شرح كل سطر:

from sqlalchemy import Column, Integer, String, DateTime, Boolean
from datetime import datetime
from app.infra.db.base import Base

class Student(Base):
    __tablename__ = "students"   # the actual name of the table in SQLite

    id = Column(Integer, primary_key=True, autoincrement=True)
    # primary_key=True  → this column uniquely identifies each row
    # autoincrement=True→ SQLite automatically assigns 1, 2, 3...

    zoho_id = Column(String, unique=True, nullable=True, index=True)
    # unique=True  → no two rows can have the same zoho_id
    # nullable=True→ this field is allowed to be empty (None)
    # index=True   → database creates a fast lookup index for this column

    moodle_id  = Column(Integer, nullable=True)
    first_name = Column(String, nullable=False)   # required — cannot be empty
    last_name  = Column(String, nullable=False)   # required
    email      = Column(String, unique=True, nullable=False)
    student_id = Column(String, nullable=True)    # academic number e.g. A01B1234C
    program    = Column(String, nullable=True)

    created_at = Column(DateTime, default=datetime.utcnow)
    # default=datetime.utcnow → automatically set to current time when INSERT runs

    updated_at = Column(DateTime, onupdate=datetime.utcnow)
    # onupdate=datetime.utcnow → automatically set to current time when UPDATE runs

When the server starts, SQLAlchemy reads all these class definitions and executes CREATE TABLE IF NOT EXISTS students (...) for you.

عند تشغيل السيرفر، SQLAlchemy يقرأ كل هذه الكلاسات وينفّذ CREATE TABLE IF NOT EXISTS students (...) تلقائياً. إذا كان الجدول موجوداً فعلاً، لا يفعل شيئاً ولا يمس البيانات الموجودة.

5. All 10 Database Tables — Purpose and Key Fields

٥. الجداول العشرة — الغرض والحقول الأساسية

📋 students — الطلاب

The central table. Links each Moodle user to their Zoho Contact. All other tables reference students. The email column is the matching key between Moodle and Zoho.

الجدول المحوري. يربط كل مستخدم في Moodle بجهة اتصاله في Zoho. جميع الجداول الأخرى تشير إليه. الإيميل هو مفتاح المطابقة بين النظامين.

📋 enrollments — التسجيلات في المواد

Records which student is enrolled in which Moodle course. Created when a Moodle enrollment event arrives via webhook, OR when synced from Zoho registrations.

يسجّل أي طالب مسجّل في أي مادة/كورس في Moodle. يُنشأ إما عند وصول حدث تسجيل من Moodle عبر webhook، أو عند المزامنة من Zoho.

يخزّن بيانات الدرجات والإكمال المجلوبة من Moodle. بعد تخزينها هنا، تقوم خدمة مزامنة الدرجات بدفعها إلى Zoho CRM.

📋 payments — المدفوعات

Fee payment records from Zoho Deals. Allows checking if a student has paid before certain actions are taken (like enrolling in a course).

سجلات مدفوعات الرسوم من وحدة Deals في Zoho. تُستخدم للتحقق من أن الطالب قد سدّد رسومه قبل تنفيذ إجراءات معينة كالتسجيل في كورس.

ColumnWhat it stores

📋 registrations — تسجيلات البرامج الدراسية

Programme-level intake registrations from Zoho. A student registers for a BTEC programme (e.g., starts September 2024). Then individual course unit enrollments are created from this.

تسجيلات الدفعة الدراسية على مستوى البرنامج من Zoho. الطالب يسجّل في برنامج BTEC (مثل: يبدأ سبتمبر 2024)، ثم تُنشأ تسجيلات الوحدات الفردية من هذا السجل.

📋 classes — الحصص / الفصول

Physical or virtual class session records from Zoho. Stores the session name, date/time schedule, and maximum capacity.

سجلات الحصص أو الجلسات (حضورية أو افتراضية) من Zoho. تخزّن اسم الجلسة، الجدول الزمني، والسعة القصوى للمجموعة.

📋 units — الوحدات التعليمية

Individual learning unit records (e.g. BTEC units). Each has a unit code, name, credit value, and links to its parent class or programme.

سجلات الوحدات التعليمية الفردية (مثل وحدات BTEC). لكل وحدة: كود، اسم، قيمة ائتمانية (credits)، ورابط للحصة أو البرنامج الأب.

📋 programs — البرامج الدراسية

Master list of qualifications the institution offers. Examples: "BTEC Level 2 Business", "BTEC Level 3 IT". Referenced by students and registration records.

القائمة الرئيسية للمؤهلات التي تقدمها المؤسسة. أمثلة: "BTEC Level 2 Business"، "BTEC Level 3 IT". يرجع إليها الطلاب وسجلات التسجيل.

📋 event_log — سجل الأحداث (Audit Trail)

The most important table for debugging. Every sync operation writes a record here — whether it succeeded or failed. If something goes wrong, this table tells you when, what, and why. Never delete records from this table.

أهم جدول للتصحيح والمراجعة. كل عملية مزامنة تكتب سجلاً هنا — سواء نجحت أم فشلت. إذا حدث خطأ ما، هذا الجدول يخبرك متى، وماذا، ولماذا. لا تحذف منه أبداً.

📋 btec_templates — قوالب BTEC

Templates defining the structure of BTEC qualifications — which units belong to which programme, their credit values, assessment criteria. Used to auto-populate unit lists when a new class is created.

قوالب تُعرِّف هيكل مؤهلات BTEC — أي وحدات تنتمي لأي برنامج، قيمها الائتمانية، ومعايير تقييمها. تُستخدم لملء قوائم الوحدات تلقائياً عند إنشاء حصة جديدة.

6. How Tables Are Created on Server Startup

٦. كيف تُنشأ الجداول عند تشغيل السيرفر

In backend/app/main.py there is a startup function that runs once when the server starts:

في backend/app/main.py يوجد كود يشتغل مرة واحدة عند بدء تشغيل السيرفر:

from contextlib import asynccontextmanager
from app.infra.db.base import Base, engine
import app.infra.db.models  # must import models so SQLAlchemy "knows" them

@asynccontextmanager
async def lifespan(app):
    # Runs ONCE right after server starts
    Base.metadata.create_all(bind=engine)
    # This reads all model classes and runs:
    # CREATE TABLE IF NOT EXISTS students (...)
    # CREATE TABLE IF NOT EXISTS enrollments (...)
    # ... for every model
    print("Database tables created/verified.")
    yield  # server is now running and handling requests

app = FastAPI(lifespan=lifespan)

create_all() uses IF NOT EXISTS — perfectly safe to call every startup. Existing tables and data are never modified.

create_all() يستخدم IF NOT EXISTS — آمن تماماً للاستدعاء في كل مرة. الجداول والبيانات الموجودة لا تُعدَّل أبداً.

You will see this in the terminal output when starting the server:

INFO:     Started server process
INFO:     database tables created/verified.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8001

7. Reading and Writing Data — Query Examples

٧. قراءة وكتابة البيانات — أمثلة على الاستعلامات

Inside endpoint functions, a SQLAlchemy Session is injected via Depends(get_db). Here are the most common operations:

داخل دوال الـ endpoints، يُحقَن session من SQLAlchemy عبر Depends(get_db). إليك أكثر العمليات شيوعاً:

from app.infra.db.models.student import Student

# ━━━ READ OPERATIONS ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# Get all students (returns a Python list)
students = db.query(Student).all()

# Filter: get students in a specific program
btec3_students = db.query(Student).filter(Student.program == "BTEC L3").all()

# Find one student by email (returns None if not found)
student = db.query(Student).filter(Student.email == "ahmad@example.com").first()

# Count total records
total = db.query(Student).count()

# Sort alphabetically by last name
sorted_students = db.query(Student).order_by(Student.last_name).all()

# Pagination (page 2, 20 per page)
page2 = db.query(Student).offset(20).limit(20).all()

# ━━━ WRITE OPERATIONS ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# Create a new student
new_student = Student(
    first_name="Ahmad",
    last_name="Ali",
    email="ahmad@example.com",
    program="BTEC L3"
)
db.add(new_student)        # stage the INSERT
db.commit()                 # write to disk
db.refresh(new_student)     # reload from DB (to get auto-generated id)
print(new_student.id)       # now has its database ID

# Update an existing student
student = db.query(Student).filter(Student.email == "ahmad@example.com").first()
student.program = "BTEC L2"   # change the field
db.commit()                    # save the change

# Delete a record (rarely done in this project — we keep history)
db.delete(student)
db.commit()

8. The Upsert Pattern — How Sync Services Work

٨. نمط الـ Upsert — كيف تعمل خدمات المزامنة

When data arrives from Zoho, it might be a new student (never seen before) or an existing one (already in the database from a previous sync). The upsert pattern handles both cases elegantly: try to find the existing record — if found, update it; if not found, create a new one.

عند وصول بيانات من Zoho، قد تكون لطالب جديد أو طالب موجود بالفعل من مزامنة سابقة. نمط الـ "upsert" يتعامل مع الحالتين: ابحث عن السجل الموجود — إن وُجد، حدِّثه؛ وإن لم يوجد، أنشئ سجلاً جديداً.

def upsert_student(db: Session, zoho_data: dict) -> Student:
    """
    Insert a new student or update an existing one.
    Uses zoho_id as the unique key to detect duplicates.
    """
    zoho_id = zoho_data["id"]  # Zoho's unique ID for this contact

    # Step 1: Search for an existing record with this zoho_id
    existing = db.query(Student).filter(Student.zoho_id == zoho_id).first()

    if existing:
        # ✅ Student already exists — update their data
        existing.first_name = zoho_data.get("First_Name", existing.first_name)
        existing.last_name  = zoho_data.get("Last_Name",  existing.last_name)
        existing.email      = zoho_data.get("Email",      existing.email)
        existing.program    = zoho_data.get("Program__c", existing.program)
        db.commit()
        db.refresh(existing)
        return existing
    else:
        # ✅ New student — create a brand new record
        new_student = Student(
            zoho_id    = zoho_id,
            first_name = zoho_data.get("First_Name"),
            last_name  = zoho_data.get("Last_Name"),
            email      = zoho_data.get("Email"),
            program    = zoho_data.get("Program__c"),
        )
        db.add(new_student)
        db.commit()
        db.refresh(new_student)
        return new_student

This same upsert logic exists in enrollment_service.py, grade_service.py, and payment_service.py. It is what makes it safe to run the same sync multiple times without creating duplicate records.

نفس منطق الـ upsert موجود في enrollment_service.py وgrade_service.py وpayment_service.py. هو ما يجعل تشغيل المزامنة أكثر من مرة آمناً تماماً دون إنشاء سجلات مكررة.

9. How to Browse the Database

٩. كيف تتصفح قاعدة البيانات

Option A: Admin Panel Data Browser (Easiest)

الأسهل: لوحة تحكم المشروع

Open http://localhost:8001/admin/data while the server is running. You will see paginated tables of students, enrollments, grades, and payments with search.

افتح http://localhost:8001/admin/data أثناء تشغيل السيرفر. ستجد جداول مُرقَّمة للطلاب والتسجيلات والدرجات والمدفوعات مع إمكانية البحث.

Option B: Python built-in sqlite3 module

import sqlite3

conn = sqlite3.connect("backend/moodle_zoho_local.db")
cursor = conn.cursor()

# List all tables
tables = cursor.execute("SELECT name FROM sqlite_master WHERE type='table'").fetchall()
print("Tables:", [t[0] for t in tables])

# Count students
count = cursor.execute("SELECT COUNT(*) FROM students").fetchone()[0]
print(f"Students in DB: {count}")

# See first 3 students
rows = cursor.execute("SELECT id, first_name, email, program FROM students LIMIT 3").fetchall()
for row in rows:
    print(row)

conn.close()

Option C: DB Browser for SQLite (Free visual GUI)

Download from https://sqlitebrowser.org (free, Windows/Mac/Linux). Open it, select "Open Database", and choose backend/moodle_zoho_local.db. You can visually browse tables, run queries, and even edit data.

حمّله من https://sqlitebrowser.org (مجاني). افتحه، اختر "Open Database"، ثم اختر ملف backend/moodle_zoho_local.db. يمكنك تصفح الجداول بشكل مرئي، تشغيل استعلامات SQL، وتعديل البيانات مباشرة. مناسب جداً للمبتدئين.
